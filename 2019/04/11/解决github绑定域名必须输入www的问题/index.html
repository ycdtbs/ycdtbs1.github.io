解决github绑定域名必须输入www的问题

发表于 2019-04-11 |

   博主在配置hexo时绑定购买的域名，在CANAME的设置正确以及所有文件配置正确的情况下，有时需要输入域名全称，造成了极大的不便，经过多次实验发现了问题所在
   我们通常在解析域名时需要添加两种类型的记录，第一种为CNAME指将一个域名解析到另一个域名，这次解析方法再这里不详细解说，问题出在第二种解析方式上，第二种解析方式为A指的是将域名解析到另一个IPV4地址，我们通常查看网页ip地址的方式是在控制台输入CMD，通过ping命令来查看我们网页的ip地址，操作如图所示

   这里我们发现了自己网页的地址，通过添加记录的方式将域名解析到了我们的网站上，可是经过测试，连接并不稳定，这是由于我们ping的网页仅仅给了我们一个IPV4的地址，我们通过另一款工具http://ip.tool.chinaz.com/

查询后的ip如下

   可以看到我们的网页有四个ip地址，所以我们只需要将剩余的三个ip地址添加到域名控制台的解析中就可以了

    操作完成后我们在通过域名访问我们的地址，发现任何时候都可以访问了
